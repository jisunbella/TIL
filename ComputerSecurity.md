# Computer Security - Overview

## 1. 컴퓨터 보안의 개요

---

## 정보보호의 개념

* 정보를 여러 가지 위협으로부터 보호하기 위한 정책 및 기법

  * 정책: option, 삭제 정책 등
  * 기법: 기술, DLP 보안 솔루션 등

* 정보의 상태

  * 저장
  * 전달 (도청 가능성 있음)

* 위협의 종류

  * 허락되지 않은 접근
  * 수정
  * 훼손
  * 유출

### Example: 금고

Plain Text + Key → Cipher Text

* 암호 알고리즘은 공개됨
* Key 관리가 매우 중요

---

## 정보보호의 목표

### 핵심 목표 (CIA Triad)

* 기밀성 Confidentiality: 승인받지 않은 공개 방지
* 무결성 Integrity: 승인받지 않은 변조 방지
* 가용성 Availability: 서비스 중단 방지

### 추가 목표

* 부인방지 Non-repudiation
* 인증 Authentication
* 접근제어 Access Control

---

## 기밀성 (Confidentiality)

* 허락되지 않은 사용자가 정보를 알 수 없도록 하는 것

### 방법

* 접근 차단
* 암호화

### 예시

* 은행 고객 정보 보호

* 개인정보는 반드시 암호화 필요

---

## 무결성 (Integrity)

* 허락되지 않은 수정 방지

### 방법

Message → Hash Function → Hash Code

* 해시코드 비교로 위변조 확인

---

## 가용성 (Availability)

* 허락된 사용자가 언제든지 정보에 접근 가능해야 함

### 방해 요소

* DDoS 공격

### 실무

* 보안 체크리스트 사용 (80~400개)

---

## 부인방지 (Non-repudiation)

* 송수신 사실을 부인하지 못하도록 하는 것

### 종류

* 발신 부인방지
* 수신 부인방지

### 실무

* 로그 기록 필수 (최소 1년 이상)

---

## 인증 (Authentication)

* 사용자의 신원을 검증

### 예시

* Password
* PIN

---

## 접근제어 (Access Control)

* 허용된 사용자만 접근 가능

* 사용자별 권한 관리

---

## 컴퓨터 보안의 역사

### 초기

* 앨런 튜링 → 컴퓨터 개념 정립
* 콜로서스 → 암호 해독

### 1950~60년대

* 메인프레임 사용
* 해커 용어 등장

### 1970년대

* 개인용 컴퓨터 등장

### 1980년대

* 인터넷 확산 (TCP/IP)
* 모리스 웜
* CERT 설립

### 1990년대

* 금융/정부 해킹 증가

### 2000년대

* DDoS 공격 증가

### 2010년대

* 랜섬웨어 등장

Decrypt = Key 필요

---

## 한 줄 정리

보안의 핵심은 데이터를 안전하게 보호하고 필요한 사람만 접근하게 하는 것
